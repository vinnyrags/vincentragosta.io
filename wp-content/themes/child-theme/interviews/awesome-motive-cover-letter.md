**Favorite WordPress hook:**

`save_post_{$post_type}` — because it's where raw content becomes structured data. On a recent project, I used `save_post_project` to automatically extract a project year from block content markup and store it as queryable post meta. It replaced a manual workflow with something invisible — the content team never thinks about it, but the frontend gets clean, reliable data. I like hooks that build infrastructure you never have to touch again. The post-type-specific variant keeps things targeted so you're not running logic against every save in the system.
