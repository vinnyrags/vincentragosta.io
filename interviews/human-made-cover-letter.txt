Where I'm coming from:

I spent nearly nine years at Situation Interactive, a digital marketing agency in New York, where I was part of a 4-person full-stack team building and maintaining 75+ web properties for clients across Broadway, performing arts, and cultural institutions — platforms serving 4M+ annual visitors with unpredictable traffic spikes around on-sale dates and opening nights. The technical overlap:

My current personal platform is built on a custom parent-child theme architecture with PHP 8.4, strict types, dependency injection via PHP-DI, Timber/Twig templating, and a provider pattern inspired by Laravel service providers. Each provider self-registers its hooks, blocks, features, assets, and tests through a composition-based architecture — it's the kind of reusable, maintainable infrastructure that scales across projects and teams. I build custom Gutenberg blocks with server-side rendering and React editor interfaces, manage dependencies through Composer and npm, and run esbuild + Sass build pipelines. I work with MySQL daily and I'm experienced with query optimization, caching strategies, and the performance considerations that come with database-driven applications at scale.

I've integrated with a wide range of RESTful APIs — Ticketmaster, Telecharge, Blackbaud, Stripe, WooCommerce, and others — and I'm comfortable building custom REST endpoints as well as consuming external services. Git and GitHub are central to my workflow, including branching strategies, pull request reviews, and CI/CD pipelines.

Quality and accessibility:

Code quality isn't something I bolt on — it's structural. One of my current projects has 430+ automated tests across PHPUnit, Vitest, and Playwright, with WCAG 2.1 AA accessibility enforcement baked into the E2E suite via axe-core. I write code with strict types, semantic markup, and accessibility built in from the start. Performance and security are baseline concerns, not cleanup passes before launch.

On AI-assisted development:

I use AI tools daily — Claude Code is central to my workflow — and I have a clear-eyed view of where they accelerate work and where human judgment needs to stay in the driver's seat. AI is excellent for research, boilerplate, stress-testing ideas, and catching blind spots. But I don't ship code I don't understand, and I don't hand over architectural decisions. The tools move faster when the developer driving them knows what good output looks like.

I'm looking for a place where I can deepen my technical expertise within a team that holds itself to a high standard, and Human Made clearly fits that description.

I'd genuinely welcome the chance to contribute to what you're building.
